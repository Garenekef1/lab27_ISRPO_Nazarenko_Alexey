1.  Minimal API обеспечивает более высокую производительность, меньший объем памяти и удобен для микросервисов
2.  Если поставить app.MapGet(...) (или любой другой метод маршрутизации эндпоинтов) до app.Use(...) в конвейере обработки запросов (middleware pipeline) ASP.NET Core, запрос будет обработан конечной точкой до того, как он пройдет через промежуточное ПО (middleware), определенное после маршрутизации.
3.  ASP.NET Core преобразует имена свойств из PascalCase (например, FirstName) в camelCase (например, firstName) при сериализации в JSON по умолчанию
4.  Коды HTTP — это стандартные ответы сервера, которые сообщают браузеру о результате запроса.
    200 OK (Успех)

        Что означает: Запрос выполнен успешно.
        Контекст: Сервер нашел страницу, обработал запрос и вернул ожидаемые данные (например, открыл сайт, загрузил картинку или выполнил команду).

401 Unauthorized (Неавторизован)

    Что означает: Пользователь не прошел аутентификацию.
    Контекст: Запрос требует, чтобы пользователь представился (ввел логин/пароль), но данные отсутствуют или неверны. Сервер знает, кто вы, но не разрешает доступ, пока вы не авторизуетесь.

404 Not Found (Не найдено)

    Что означает: Запрашиваемый ресурс не найден на сервере.
    Контекст: Самая частая ошибка: страница удалена, перемещена, или вы ошиблись в адресе (URL). Сервер работает, но страницы по этому адресу нет.

5. dotnet run просто запускает приложение один раз, а dotnet watch run отслеживает изменения в файлах и автоматически перезапускает его или применяет Hot Reload (горячую перезагрузку) при сохранении кода. Первое подходит для обычного запуска, второе — для активной разработки
